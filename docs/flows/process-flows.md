# Process Flows

## Process 1: Foundation Pack Lifecycle (MVP)
1. Intake SRS.
2. Run ambiguity scan.
3. Draft product artifacts (brief/priorities/scope).
4. Draft architecture and design artifacts.
5. Draft API, acceptance matrix, and test strategy.
6. Collect unresolved assumptions in open questions.
7. Cross-artifact consistency review.
8. Approve or return for revision.
9. Mark package ready for backlog generation.

## Process 2: Change Management
1. Stakeholder requests change.
2. Impact analysis across all Foundation Pack artifacts.
3. Update affected documents.
4. Re-run acceptance and test alignment checks.
5. Re-approve changed artifact set.

## Process 3: Pre-Implementation Readiness Gate
1. Verify all mandatory artifacts exist.
2. Verify MVP-vs-later-phase separation.
3. Verify open questions either resolved or explicitly accepted.
4. Verify acceptance matrix to test strategy mapping.
5. Verify API contracts include examples and error shapes.
6. Authorize creation of implementation task packages.

## Ambiguities Identified
- Approval authority and escalation path are undefined.
- Required turnaround times for each process step are unknown.
