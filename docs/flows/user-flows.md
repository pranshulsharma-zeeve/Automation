# User Flows

## Flow 1: Create Foundation Pack from SRS (MVP)
1. User opens project workspace.
2. User uploads or references SRS.
3. System prompts for missing mandatory context.
4. User drafts product brief, priorities, and scope.
5. User completes architecture and flow artifacts.
6. User completes design artifacts and API contract.
7. User fills acceptance matrix and test strategy.
8. User logs unresolved assumptions in open questions.
9. System runs readiness checks and highlights gaps.
10. User resolves gaps and submits for review.

## Flow 2: Review & Approve Artifact Set (MVP)
1. Reviewer opens artifact checklist.
2. Reviewer validates consistency across docs.
3. Reviewer marks issues/questions inline.
4. Author updates artifacts.
5. Reviewer re-checks and approves.
6. System marks Foundation Pack as "Ready for backlog generation".

## Flow 3: Clarification Loop (MVP)
1. User encounters ambiguity while authoring.
2. User creates open question entry.
3. Owner responds with clarification.
4. User updates corresponding artifact.
5. System records closure and traceability.

## Later-Phase User Flows
- One-click task-package generation from approved artifacts.
- Integration-driven artifact sync with PM tools.
